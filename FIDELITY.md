# GraphWar 经典规则对照

本项目的经典规则依据 GraphWar 1 官方教程与 GPL 源码实现；界面、美术和声音采用原创网页实现，不复制原作素材。

## 已对照的权威规则

| 原版规则/常量 | MyGraphWar 实现 |
| --- | --- |
| 770×450 绘图区，x 轴长度 50 | 相同内部 Canvas 尺寸与 `x=-25…25`、`y≈±14.6104` 映射 |
| 普通函数、一阶 ODE、二阶 ODE | 三种经典模式，普通函数纵向平移，ODE 使用 RK4 |
| 第二阶 ODE 使用士兵独立角度 | 每名士兵保存角度，上下键及滑杆同步服务器 |
| 步长 0.01、最大 20000 点、自适应距离 0.001 | 相同采样上限、基础步长和自适应阈值 |
| 每秒绘制 1500 点、完成后淡出 1 秒 | 相同播放速率与淡出时间 |
| 轨迹完成后延迟 3 秒换人 | 服务器结算为 `点数/1500 + 3 秒` |
| 60 秒回合、5 秒开局延迟 | 服务器权威计时与同步截止时间 |
| 最多 10 人、每人默认 2/最多 4 名士兵 | 经典房间使用相同限制 |
| A/B 两队、左右半场出生、开局交替重排 | 相同队伍、出生和随机先手逻辑 |
| 约 15 个随机圆形障碍，均值半径 40 px | 相同高斯数量/半径模型及坐标换算 |
| 士兵半径 7 px、爆炸缺口半径 12 px | 相同像素比例换算与碰撞半径 |
| 函数可连续命中士兵，撞障碍停止 | 相同直接击杀与障碍终止行为 |
| `-skip`、`-sayfunc`、`-shownext` 命令 | 命令及对应停止命令均实现 |
| 5 秒保活、30 秒断开 | 浏览器心跳、服务器超时、重连与 AI 托管 |
| 玩家独立颜色且不能太亮 | 相同颜色模长上限，颜色用于士兵和函数 |

## 有意保留的差异

- 浏览器账号、SQLite 战绩、观战、重放和自动重连是新增能力。
- 扩展规则允许最多 20 人、自由混战、极坐标和参数方程；不会出现在经典规则中。
- 美术和音效为原创实现，避免直接复制原作资源。
- 原版 Java TCP 协议替换为版本化 REST/WebSocket JSON 协议，游戏判定仍由服务器权威执行。
